Overview

Docker uses OS-level virtualization to run containers. Images are layered, immutable templates; containers are runnable instances. Compose orchestrates multi-container apps. Registries (like Docker Hub) store images.

Isolation

Namespaces and cgroups isolate processes and resources for reliable, reproducible environments.

Layers

Copy-on-write union filesystem makes image layers efficient and cacheable during builds.

Networking

Bridge networks connect containers; publish ports to host for external access.

Dockerfile

# Dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3000
CMD ["node","server.js"]

Docker Compose

services:
  api:
    build: .
    ports: ["3000:3000"]
    environment:
      - DATABASE_URL=postgres://postgres:postgres@db:5432/app
    depends_on: [db]
  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_PASSWORD=postgres
      - POSTGRES_DB=app
    volumes:
      - pg:/var/lib/postgresql/data
volumes:
  pg: {}

Essential Commands

# Build and run
docker build -t app:dev .
docker run --rm -p 3000:3000 app:dev

# Compose
docker compose up -d

# Inspect
docker ps
docker logs -f <container>
docker exec -it <container> sh

# Cleanup
docker compose down
docker system prune -f
