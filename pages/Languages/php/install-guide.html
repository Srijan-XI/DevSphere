Windows Installation

Option A: XAMPP (Easiest ‚Äì Includes Apache + MySQL + PHP)

  1. Download XAMPP installer for Windows.
  2. Run the installer and choose components (Apache, MySQL, PHP, phpMyAdmin).
  3. Install to C:\xampp (default).
  4. Launch XAMPP Control Panel and start Apache.
  5. Test at http://localhost in your browser.
Note: XAMPP is for development only ‚Äì not recommended for production servers.

Option B: Standalone PHP Installation

  1. Download PHP from windows.php.net/download.
  2. Choose "Thread Safe" version (for Apache) or "Non-Thread Safe" (for IIS/Nginx).
  3. Extract to C:\php.
  4. Add C:\php to System PATH.
  5. Copy php.ini-development to php.ini.

Add PHP to PATH (PowerShell Admin):

[System.Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\php", "Machine")

Option C: Chocolatey Package Manager

# Install Chocolatey first (if not installed)
Set-ExecutionPolicy Bypass -Scope Process -Force
iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

# Install PHP
choco install php -y

# Verify
php -v

macOS Installation

Option A: Homebrew (Recommended)

# Install Homebrew if missing
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install PHP (latest version)
brew install php

# Or install specific version
brew install php@8.3

# Start PHP service
brew services start php

# Verify
php -v

Option B: MacPorts

sudo port install php83
sudo port select --set php php83

Option C: MAMP (Apache + MySQL + PHP Bundle)

  1. Download MAMP for macOS.
  2. Install and launch MAMP.
  3. Start servers from MAMP control panel.
  4. Access at http://localhost:8888.

Configure php.ini Location

php --ini  # Find loaded config file path

Linux Installation

Debian/Ubuntu

sudo apt update
sudo apt install php php-cli php-fpm php-mysql php-xml php-mbstring -y

# Verify
php -v

Install Specific Version (e.g., PHP 8.3):

sudo apt install software-properties-common -y
sudo add-apt-repository ppa:ondrej/php -y
sudo apt update
sudo apt install php8.3 php8.3-cli php8.3-fpm php8.3-mysql -y

Fedora/RHEL/CentOS

sudo dnf install php php-cli php-fpm php-mysqlnd php-xml php-mbstring -y

# Start PHP-FPM
sudo systemctl start php-fpm
sudo systemctl enable php-fpm

Arch Linux

sudo pacman -S php php-fpm

Composer Installation

Composer is the dependency manager for PHP (like npm for Node.js).

Windows

  1. Download Composer-Setup.exe.
  2. Run installer (will auto-detect PHP).
  3. Verify: composer --version

macOS/Linux

php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php composer-setup.php
php -r "unlink('composer-setup.php');"
sudo mv composer.phar /usr/local/bin/composer

# Verify
composer --version

Using Composer

# Create new project
composer create-project laravel/laravel my-app

# Install dependencies
composer install

# Add package
composer require monolog/monolog

# Update packages
composer update

Web Server Configuration

Built-in Development Server

# Simple server
php -S localhost:8000

# With specific document root
php -S localhost:8000 -t public/

# With router script
php -S localhost:8000 router.php

Apache Configuration

Enable PHP module in httpd.conf or apache2.conf:

LoadModule php_module modules/libphp.so
AddHandler php-script .php
DirectoryIndex index.php index.html

Nginx + PHP-FPM

Configure Nginx site (/etc/nginx/sites-available/default):

server {
    listen 80;
    server_name localhost;
    root /var/www/html;
    index index.php index.html;

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}

Restart Nginx: sudo systemctl restart nginx

Best Practices

  • ‚úÖ Use PHP 8.0+ for modern features and performance.
  • ‚úÖ Enable display_errors=Off in production.
  • ‚úÖ Use error_reporting(E_ALL) in development.
  • ‚úÖ Enable OPcache for better performance.
  • ‚úÖ Use environment variables for sensitive config (via .env files).
  • ‚úÖ Follow PSR coding standards (PSR-1, PSR-12).

  • ‚úÖ Validate and sanitize all user inputs.
