What is MySQL?

MySQL is a robust, open-source RDBMS powering millions of applications. It supports ACID transactions, strong indexing, and replication for high availability.

  • Storage engine: InnoDB (default) with MVCC, row-level locking.
  • Replication: Async/semisync; Group Replication for HA.
  • Schemas: Structured, normalized data in tables, rows, columns.

Core Concepts

  • Databases & tables organize data in a structured schema.
  • Primary/foreign keys enforce identity and relationships.
  • Indexes accelerate lookups; use covering and composite indexes wisely.
  • Transactions ensure atomicity, consistency, isolation, durability.

SQL Examples

Create database & user

CREATE DATABASE appdb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'app'@'%' IDENTIFIED BY 'StrongP@ssw0rd';
GRANT ALL PRIVILEGES ON appdb.* TO 'app'@'%';
FLUSH PRIVILEGES;

Create table

USE appdb;
CREATE TABLE users (
	id BIGINT PRIMARY KEY AUTO_INCREMENT,
	email VARCHAR(255) NOT NULL UNIQUE,
	name VARCHAR(120) NOT NULL,
	created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
	KEY idx_users_created_at (created_at)
) ENGINE=InnoDB;

CRUD

INSERT INTO users(email,name) VALUES ('ada@example.com','Ada');
SELECT id,email,name FROM users WHERE email='ada@example.com';
UPDATE users SET name='Ada L.' WHERE email='ada@example.com';
DELETE FROM users WHERE email='ada@example.com';

Performance

  • Prefer covering indexes for frequently queried columns.
  • Use EXPLAIN to analyze query plans.
  • Avoid SELECT *; project only needed columns.
  • Normalize to reduce duplication; denormalize selectively for reads.
