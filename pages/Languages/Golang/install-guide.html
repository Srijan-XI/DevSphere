Windows Installation

Option A: Official Installer (Recommended)

  1. Download from go.dev/dl/ ‚Äì choose Windows MSI installer.
  2. Run installer (e.g., go1.21.5.windows-amd64.msi).
  3. Accept default installation path: C:\Program Files\Go
  4. Installer automatically sets PATH and GOPATH.

Restart terminal to apply changes.

Option B: Chocolatey

choco install golang -y

Option C: Scoop

scoop install go

Manual Environment Variables (if needed)

# PowerShell (Admin)
[System.Environment]::SetEnvironmentVariable("GOPATH", "$HOME\go", "User")
[System.Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\Go\bin;$HOME\go\bin", "User")
Note: Restart terminal after PATH changes to apply settings.

macOS Installation

Option A: Official Installer

  1. Download from go.dev/dl/ ‚Äì choose macOS PKG installer.
  2. Open package and follow installation wizard.
  3. Default install: /usr/local/go

Option B: Homebrew (Recommended)

brew install go

Set Environment Variables (Add to ~/.zshrc or ~/.bash_profile)

export GOPATH=$HOME/go
export PATH=$PATH:/usr/local/go/bin:$GOPATH/bin

Reload shell: source ~/.zshrc

Note: With Go 1.11+, modules are default; GOPATH is less critical but still used for installed binaries.

Linux Installation

Option A: Official Tarball (Recommended)

  1. Download latest version:
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz
  1. Remove old installation and extract:
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
  1. Add to PATH (edit ~/.bashrc or ~/.profile):
export PATH=$PATH:/usr/local/go/bin
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin
  1. Reload: source ~/.bashrc

Option B: Package Manager

Debian/Ubuntu (may be outdated):

sudo apt update
sudo apt install golang-go -y

Fedora:

sudo dnf install golang -y

Arch:

sudo pacman -S go
Warning: Distro packages may lag behind official releases. Use tarball for latest version.

Verification

go version
go env GOPATH
go env GOROOT

Test Installation

Create hello.go:

package main

import "fmt"

func main() {
    fmt.Println("Go is installed!")
}

Run directly:

go run hello.go

Or build executable:

go build hello.go
./hello  # Linux/macOS
hello.exe # Windows
Expected Output: Go is installed!

Go Modules (Modern Approach)

Create a new project:

mkdir myproject && cd myproject
go mod init github.com/yourusername/myproject

This creates go.mod file for dependency management.

Essential Tools

Formatting & Linting

# Format code (standard style)
go fmt ./...

# Vet for common issues
go vet ./...

# Install golangci-lint (meta-linter)
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest
golangci-lint run

Testing

# Run all tests
go test ./...

# With coverage
go test -cover ./...

# Verbose output
go test -v ./...

# Benchmarks
go test -bench=.

Build & Install

# Build executable
go build -o myapp

# Build for different platforms
GOOS=linux GOARCH=amd64 go build -o myapp-linux
GOOS=windows GOARCH=amd64 go build -o myapp.exe

# Install to $GOPATH/bin
go install

Popular Tools

# gopls (language server for IDEs)
go install golang.org/x/tools/gopls@latest

# delve (debugger)
go install github.com/go-delve/delve/cmd/dlv@latest

# staticcheck (linter)
go install honnef.co/go/tools/cmd/staticcheck@latest

Best Practices

  • Use Go Modules: Always init with go mod init.
  • Format automatically: Run gofmt or go fmt before commits.
  • Follow conventions: Package names lowercase, no underscores.
  • Error handling: Check every error explicitly.
  • Testing: Write tests in *_test.go files.
  • Documentation: Use godoc comments (// Package pkg ...).
  • Vendoring: Use go mod vendor for offline builds.
IDE Setup

VS Code:

  1. Install VS Code.
  2. Install "Go" extension by Go Team at Google.
  3. Install recommended tools when prompted (gopls, dlv, etc.).

GoLand: JetBrains IDE with full Go support (commercial).

Vim/Neovim: Install vim-go plugin or use LSP with gopls.

Environment Variables

GOROOT: Go installation directory (set automatically).

GOPATH: Workspace for installed binaries and cached modules (default: ~/go).

GOBIN: Directory for installed executables (default: $GOPATH/bin).

GO111MODULE: Enable modules (default: on since Go 1.16).

GOPROXY: Module proxy (default: https://proxy.golang.org).

View all: go env

Cross-Compilation

# Windows AMD64
GOOS=windows GOARCH=amd64 go build -o app.exe

# macOS ARM64 (M1/M2)
