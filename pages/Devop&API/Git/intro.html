Git — Introduction

Distributed version control for individuals and teams. Stage, commit, branch, and collaborate confidently.

DVCS BRANCHES REBASE MERGE
Concepts Workflow Branching Merge/Rebase Commands

Core Concepts

  • Repository: A project history tracked by Git.
  • Working tree, index (staging area), and HEAD.
  • Commit: Snapshot with metadata and parent links.
  • Remotes: References to repositories (e.g., origin on GitHub).

Basic Workflow

git clone https://github.com/user/repo.git
cd repo
git checkout -b feature/login
# edit files
git add .
git commit -m "feat: add login"
git push -u origin feature/login

Branching Strategy

Use short-lived feature branches; protect main with PR reviews and CI. Keep history readable with conventional commits.

# List and create branches
git branch -a
git checkout -b fix/typo

Merge vs Rebase

# Merge: preserves branch structure
git checkout main
git merge feature/login

# Rebase: linear history
git checkout feature/login
git rebase main

Common Commands

git status
git diff
git add -p
git commit -m "chore: ..."

# Undo
git restore --staged file
git checkout -- file

# Remote
git remote -v
git pull --rebase
git push
