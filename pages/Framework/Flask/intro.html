Overview

Flask is a micro-framework for Python that provides routing, templating, and request handling. It's unopinionated and extensible via a rich ecosystem of extensions.

from flask import Flask
app = Flask(__name__)

@app.route('/')
def index():
    return 'Hello Flask'

if __name__ == '__main__':
    app.run(debug=True)

Routing

Define routes with decorators. Support dynamic segments, query params, and HTTP methods.

@app.route('/items')
def list_items():
    return {'items': []}

@app.route('/items/', methods=['GET', 'POST'])
def item_detail(item_id):
    if request.method == 'POST':
        return {'created': item_id}, 201
    return {'id': item_id}
from flask import request

@app.route('/search')
def search():
    q = request.args.get('q', '')
    return {'query': q}

Templates

Use Jinja2 for rendering HTML templates with variables, loops, filters, and inheritance.

# templates/base.html
<html>
  <head><title>{% block title %}App{% endblock %}</title></head>
  <body>{% block content %}{% endblock %}</body>
</html>

# templates/index.html
{% extends "base.html" %}
{% block content %}
  {% for item in items %}
    <p>{{ item.name }}</p>
  {% endfor %}
{% endblock %}
from flask import render_template

@app.route('/')
def index():
    return render_template('index.html', items=[])

Blueprints

Organize routes into blueprints for modular apps. Register blueprints to the main app with URL prefixes.

from flask import Blueprint

api = Blueprint('api', __name__, url_prefix='/api')

@api.route('/items')
def items():
    return {'items': []}

# In main app
app.register_blueprint(api)

Extensions

Flask extensions add functionality like ORM (Flask-SQLAlchemy), auth (Flask-Login), forms (Flask-WTF), and REST (Flask-RESTful).

# Example: Flask-SQLAlchemy
from flask_sqlalchemy import SQLAlchemy
db = SQLAlchemy(app)

class Item(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100))

Common Pitfalls

  • Debug mode in production: Never run app.run(debug=True) in production; use a WSGI server like Gunicorn.
  • Missing SECRET_KEY: Set a strong SECRET_KEY in config for sessions and CSRF protection.
  • Not using app factory: Use create_app() pattern for testing and configuration flexibility.
  • Circular imports: Move imports inside functions or use blueprints to avoid import loops.
  • No request context: request, session, g only work inside request handlers; use with app.app_context() for CLI.
