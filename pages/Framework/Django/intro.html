Overview

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It includes an ORM, template engine, admin panel, and authentication out of the box.

# Quick start
django-admin startproject mysite
cd mysite
python manage.py runserver

MTV Pattern

Django uses Model-Template-View (MTV): Models define data, Templates render HTML, and Views handle logic and return responses.

# Project structure

  mysite/

URLs & Views

Map URL patterns to view functions or classes. Views process requests and return HttpResponse or render templates.

# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index'),
    path('item//', views.detail, name='detail'),
]
# views.py
from django.shortcuts import render
from django.http import JsonResponse

def index(request):
    return render(request, 'index.html', {'items': []})

def detail(request, pk):
    return JsonResponse({'id': pk})

Models & ORM

Define database schema with Python classes. Django's ORM translates Python to SQL and supports migrations.

# models.py
from django.db import models

class Item(models.Model):
    name = models.CharField(max_length=200)
    price = models.DecimalField(max_digits=10, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name
# Migrations
python manage.py makemigrations
python manage.py migrate

# Query
Item.objects.all()
Item.objects.filter(price__gt=10)
Item.objects.create(name='Widget', price=9.99)

Templates

Use Django Template Language (DTL) with variables, filters, and tags. Extend base templates for consistency.

<!-- base.html -->
<html>
  <head><title>{% block title %}App{% endblock %}</title></head>
  <body>{% block content %}{% endblock %}</body>
</html>

<!-- index.html -->
{% extends "base.html" %}
{% block content %}
  {% for item in items %}
    <p>{{ item.name }} - ${{ item.price }}</p>
  {% endfor %}
{% endblock %}

Admin Panel

Automatic CRUD interface for models. Register models in admin.py and create a superuser.

# admin.py
from django.contrib import admin
from .models import Item

@admin.register(Item)
class ItemAdmin(admin.ModelAdmin):
    list_display = ('name', 'price', 'created_at')
    search_fields = ('name',)
python manage.py createsuperuser
# Visit http://localhost:8000/admin

Common Pitfalls

  • Forgot makemigrations: Run python manage.py makemigrations after model changes before migrate.
  • Missing INSTALLED_APPS: Add new apps to INSTALLED_APPS in settings.py.
  • DEBUG=True in production: Always set DEBUG=False and configure ALLOWED_HOSTS in production.
  • N+1 queries: Use select_related() and prefetch_related() to optimize queries.
  • Hardcoding URLs: Use {% url 'name' %} in templates and reverse('name') in views instead of hardcoding paths.
