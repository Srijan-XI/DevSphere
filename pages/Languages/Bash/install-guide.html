Note: On Windows, native cmd.exe and PowerShell differ from Bash—use WSL or Git Bash for closest parity.

Windows Installation

Option A: Git Bash (Quick)

  1. Download Git for Windows.
  2. Run installer & choose "Git Bash Here" context menu.
  3. Pick default editor (VS Code recommended).
  4. Use bundled OpenSSH & default line endings (LF).
  5. Finish and open Git Bash from Start Menu.

Option B: Windows Subsystem for Linux (WSL)

  1. Enable WSL feature (Administrator PowerShell):
wsl --install
  1. Reboot when prompted.
  2. Launch Ubuntu (or chosen distro) and create user.
  3. Update packages:
sudo apt update && sudo apt upgrade -y

WSL offers near-native Linux experience—ideal for tooling parity.

macOS Installation

macOS ships with Bash (older version) and Zsh as default shell in new releases. You can install a newer Bash using Homebrew.

  1. Install Homebrew if missing:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  1. Install latest Bash:
brew install bash
  1. Add new shell to allowed list:
echo "/opt/homebrew/bin/bash" | sudo tee -a /etc/shells
  1. Change default shell:
chsh -s /opt/homebrew/bin/bash

Open a new terminal window and verify version.

Linux Installation / Update

Most Linux distributions include Bash by default. To ensure you have the latest stable version or reinstall:

Debian/Ubuntu

sudo apt update
sudo apt install --reinstall bash

Fedora

sudo dnf install bash

Arch

sudo pacman -S bash

If Bash is missing (rare), installing the package restores it.

Verification

Confirm version and default shell after installation:

bash --version
echo $SHELL

Shebang for scripts:

#!/usr/bin/env bash
set -Eeuo pipefail
echo "Script started"

Configuration Basics

Profile Files

  • ~/.bashrc – Interactive non-login shells.
  • ~/.bash_profile / ~/.profile – Login shells (macOS often uses .bash_profile).
  • ~/.inputrc – Readline keybindings.

Sample Enhancements

# ~/.bashrc additions
export HISTCONTROL=ignoredups:erasedups
export HISTSIZE=5000
export PROMPT_COMMAND='history -a'
alias ll='ls -alF'
alias gs='git status -sb'
PS1='\u@\h:\w$ '
Tip: Keep aliases version-controlled by sourcing a file in your dotfiles repo (e.g. source ~/dotfiles/aliases.sh).

Best Practices

  • Begin scripts with: #!/usr/bin/env bash
  • Enable stricter mode: set -Eeuo pipefail
  • Quote variables: "$VAR" unless intentional globbing needed.
  • Prefer $(command) over legacy backticks.
  • Avoid parsing complex data with pure Bash—use jq, python, etc.
  • Check exit codes; fail fast for CI reliability.
  • Use functions for reuse; keep scripts modular.
  • Document parameters and expected environment variables.
Security Tips

Avoid eval: It can execute untrusted code.

Sanitize inputs: Validate user-provided paths & arguments.

Use absolute paths: Prevent ambiguity and PATH hijacking.

Least privilege: Only elevate with sudo when required.

Validate downloads: Use checksums or signatures (e.g. sha256sum).
