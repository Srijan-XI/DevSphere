Windows Installation

Option A: MinGW-w64 (GCC on Windows)

  1. Download from mingw-w64.org or use WinLibs standalone builds.
  2. Extract archive to C:\mingw64
  3. Add to PATH: C:\mingw64\bin
# Verify installation (PowerShell/CMD)
gcc --version
g++ --version

Option B: MSVC (Microsoft Visual C++)

  1. Download Visual Studio 2022 Community (free).
  2. During install, select "Desktop development with C++" workload.
  3. Launch Developer Command Prompt or Developer PowerShell.
# Verify (Developer Command Prompt)
cl
nmake

Option C: Clang/LLVM

  1. Download from LLVM releases.
  2. Run installer and add to PATH.
clang --version
clang++ --version

Option D: WSL (Windows Subsystem for Linux)

Install Ubuntu/Debian via WSL, then follow Linux instructions below.

wsl --install
# After reboot, inside WSL:
sudo apt update && sudo apt install build-essential -y
PATH Configuration: After installing MinGW/Clang, restart terminal to apply PATH changes.

macOS Installation

Xcode Command Line Tools (Clang)

macOS ships with Clang compiler via Xcode Command Line Tools:

xcode-select --install

Follow prompts to install. Verify:

clang --version
clang++ --version

GCC via Homebrew

For true GCC (not Clang alias):

  1. Install Homebrew if missing:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  1. Install GCC:
brew install gcc
  1. Use explicit version (e.g., gcc-13, g++-13):
gcc-13 --version
g++-13 --version
Note: macOS gcc command is symlinked to Clang by default. Use gcc-13 for actual GCC.

Linux Installation

Debian/Ubuntu

sudo apt update
sudo apt install build-essential -y

This installs GCC, G++, Make, and essential headers.

Fedora/RHEL/CentOS

sudo dnf groupinstall "Development Tools" -y
sudo dnf install gcc gcc-c++ make -y

Arch Linux

sudo pacman -S base-devel

Install Clang (Alternative)

Debian/Ubuntu:

sudo apt install clang -y

Fedora:

sudo dnf install clang -y

Arch:

sudo pacman -S clang

Verification

Test C Compiler

Create test.c:

#include <stdio.h>

int main() {
    printf("C compiler works!\n");
    return 0;
}

Compile and run:

gcc test.c -o test
./test

Test C++ Compiler

Create test.cpp:

#include <iostream>

int main() {
    std::cout << "C++ compiler works!" << std::endl;
    return 0;
}

Compile and run:

g++ test.cpp -o testcpp
./testcpp

Check Compiler Versions

gcc --version
g++ --version
clang --version
clang++ --version

Build Tools & Systems

Make

Included with GCC toolchain. Example Makefile:

CC = gcc
CFLAGS = -Wall -O2

all: program

program: main.c
	$(CC) $(CFLAGS) main.c -o program

clean:
	rm -f program

CMake (Modern Build System)

Install CMake:

Windows: Download from cmake.org

macOS: brew install cmake

Linux: sudo apt install cmake (Debian/Ubuntu)

Sample CMakeLists.txt:

cmake_minimum_required(VERSION 3.20)
project(MyProject)

set(CMAKE_CXX_STANDARD 17)

add_executable(myapp main.cpp)

Build with CMake:

mkdir build && cd build
cmake ..
cmake --build .

Ninja (Fast Build Tool)

# Install
sudo apt install ninja-build  # Linux
brew install ninja            # macOS

# Use with CMake
cmake -G Ninja ..
ninja

VS Code Setup (Quick)

  1. Install VS Code
  2. Install "C/C++" extension (ms-vscode.cpptools)
  3. Install "CMake Tools" extension (optional)
  4. Configure c_cpp_properties.json for IntelliSense
Tip: Use VS Code tasks.json to configure build commands with F5 debug support.

Best Practices

  • Enable warnings: -Wall -Wextra -Wpedantic
  • Use optimization flags: -O2 for release, -g for debug
  • Modern C++ standards: -std=c++17 or -std=c++20
  • Static analysis: Use clang-tidy, cppcheck
  • Memory safety: Run with Valgrind, AddressSanitizer (-fsanitize=address)
  • Version control: Commit often, use .gitignore for build artifacts
  • Cross-platform builds: Prefer CMake over platform-specific makefiles
Advanced Compiler Flags

Performance:

-O3 -march=native -flto

Debug with sanitizers:

-g -fsanitize=address -fsanitize=undefined

Threading:

-pthread

Position-independent code:

-fPIC
Debugging Tools

GDB (GNU Debugger):

gdb ./program
(gdb) break main
(gdb) run
(gdb) next
(gdb) print variable

Valgrind (Memory Leak Detection):

valgrind --leak-check=full ./program

LLDB (LLVM Debugger):

lldb ./program
(lldb) b main
(lldb) run
