Overview

Next.js brings server-first React with file-system routing, data fetching on the server, and automatic code splitting. The App Router (app/) uses React Server Components for great performance and DX.

# Create a new app
npx create-next-app@latest my-app
cd my-app
npm run dev

App Router

Routing

Folders become routes. Use [param] for dynamic segments and generateStaticParams for SSG. Nested layouts provide shared UI.

API Routes

Create route handlers under app/api/*/route.ts with HTTP method exports.

// app/api/hello/route.ts
export async function GET() {
  return Response.json({ message: 'Hello' });
}

export async function POST(req: Request) {
  const body = await req.json();
  return Response.json({ ok: true, body });
}

Deployment

Build with next build and start with next start. Deploy easily to Vercel or any Node host. Use .env.local for secrets and NEXT_PUBLIC_* for client-safe envs.

npm run build
npm run start
# or
pnpm build && pnpm start

Docker Dev Environment

# Dockerfile
FROM node:20-alpine AS base
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
EXPOSE 3000
CMD ["npm", "run", "dev"]

# docker-compose.yml
services:
  nextjs:
    build:
      context: .
      target: base
    ports:
      - "3000:3000"
    volumes:
      - .:/app
      - /app/node_modules
      - /app/.next
    environment:
      - NODE_ENV=development

Common Pitfalls

  • Server/Client Component confusion: Server Components can't use hooks or browser APIs; add "use client" directive when needed.
  • Hydration mismatches: Ensure server and client render identical HTML; avoid Date.now() or Math.random() in RSC.
  • Fetch caching: Next.js caches fetch by default; use { cache: 'no-store' } or revalidate for dynamic data.
  • Layout re-renders: Layouts don't re-render on navigation; use client components for state that needs to reset.
  • Missing metadata exports: Use export const metadata in pages/layouts for SEO instead of <Head>.
