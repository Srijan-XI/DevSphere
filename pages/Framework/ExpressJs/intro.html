Overview

Express.js is a minimal, unopinionated framework for building web servers and APIs on Node.js. Its core is a middleware pipeline and a powerful routing system.

# Install & start (quick demo)
npm init -y && npm i express
echo "const e=require('express');const app=e();app.get('/',(req,res)=>res.send('Hello'));app.listen(3000)" > index.js
node index.js

Middleware

Middleware are functions with signature (req, res, next). They run in order and can modify the request/response or pass control using next().

const express = require('express');
const app = express();

// Built-in JSON body parser
app.use(express.json());

// Logger middleware
app.use((req, res, next) => {
  const start = Date.now();
  res.on('finish', () => {
    console.log(`${req.method} ${req.url} -> ${res.statusCode} ${Date.now()-start}ms`);
  });
  next();
});

Routing

Declare handlers per HTTP method and path. Group routes with express.Router() and mount under a base path.

const router = require('express').Router();
router.get('/items', (req, res) => res.json([{ id: 1 }]));
router.post('/items', (req, res) => res.status(201).json({ ok: true }));
router.get('/items/:id', (req, res) => res.json({ id: req.params.id }));
app.use('/api', router);

Params & Query

Access dynamic segments via req.params and query string via req.query. Validate inputs before use.

app.get('/search', (req, res) => {
  const { q = '', page = 1 } = req.query;
  res.json({ q, page: Number(page) });
});

app.get('/users/:userId', (req, res) => {
  const { userId } = req.params;
  res.json({ userId });
});

Error Handling

Define a final error handler with four args (err, req, res, next). Call next(err) to delegate.

// 404 (after all routes)
app.use((req, res) => res.status(404).json({ error: 'Not found' }));

// Error handler
app.use((err, req, res, next) => {
  console.error(err);
  res.status(err.status || 500).json({ error: err.message || 'Server error' });
});

Req/Res

Common response helpers: res.status, res.json, res.send, res.redirect. Read body via req.body with parsers.

app.post('/login', (req, res) => {
  const { email } = req.body;
  if (!email) return res.status(400).json({ error: 'email required' });
  res.status(200).json({ ok: true });
});

Common Pitfalls

  • Middleware order matters: JSON parser must come before route handlers; error handlers must be last.
  • Forgot next(): Middleware must call next() or send a response, or the request hangs.
  • No error handling: Use try/catch in async handlers and pass errors to next(err).
  • Exposing stack traces: Don't send err.stack in production; log it server-side only.
  • Router mounting: Remember app.use('/api', router) prefixes all router paths with /api.
