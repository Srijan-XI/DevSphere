Data Model

MongoDB stores JSON-like BSON documents in collections. Schemas are flexible; documents in a collection need not share fields. Embedding and referencing are both valid modeling techniques depending on access patterns.

{
  _id: ObjectId("..."),
  name: "Ada",
  emails: ["a@example.com", "ada@work.com"],
  profile: { title: "Engineer", org: "DevSphere" }
}

Queries

// Insert
db.users.insertOne({ name: "Ada", age: 30 })

// Find
db.users.find({ age: { $gte: 21 } }, { name: 1 })

// Update (upsert)
db.users.updateOne({ name: "Ada" }, { $set: { role: "admin" } }, { upsert: true })

// Delete
db.users.deleteOne({ name: "Ada" })

Aggregation

db.orders.aggregate([
  { $match: { status: "PAID" } },
  { $unwind: "$items" },
  { $group: { _id: "$items.sku", qty: { $sum: "$items.qty" } } },
  { $sort: { qty: -1 } }
])

Indexes

// Single field
db.users.createIndex({ email: 1 }, { unique: true })

// Compound
db.events.createIndex({ org: 1, ts: -1 })

// Text index
db.articles.createIndex({ content: "text" })

Use Cases

  • Content management and user profiles
  • Event logging and analytics
  • Catalogs with flexible attributes
  • Real-time apps with change streams
