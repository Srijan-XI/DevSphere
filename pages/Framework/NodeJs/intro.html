Node.js — Introduction

A JavaScript runtime built on Chrome's V8 engine with an event-driven, non-blocking I/O model ideal for scalable servers and tooling.

Overview

Node.js uses a single-threaded event loop with libuv to handle concurrent I/O efficiently. It powers web servers, CLIs, build tools, and microservices.

  • Non-blocking I/O via callbacks, Promises, and async/await
  • Rich package ecosystem via npm and pnpm/yarn
  • Runs on Windows, macOS, and Linux

Modules: ESM vs CommonJS

// ESM (package.json: { "type": "module" })
export function add(a, b) { return a + b }
import { add } from './math.js'

// CommonJS
module.exports = { add }
const { add } = require('./math')

HTTP Server

// server.js
import http from 'node:http'
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'application/json' })
  res.end(JSON.stringify({ ok: true }))
})
server.listen(3000)

Common Pitfalls

  • Blocking the event loop: Avoid CPU-intensive sync operations; use worker threads or offload to services.
  • ESM vs CJS mismatch: Ensure package.json has "type": "module" for ESM or use .mjs/.cjs extensions.
  • Missing error handling: Always handle Promise rejections and use try/catch with async/await.
  • Callback hell: Prefer Promises and async/await over deeply nested callbacks.
  • Not setting NODE_ENV: Use NODE_ENV=production for optimizations in frameworks like Express.
