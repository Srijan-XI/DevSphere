JavaScript + Types: TypeScript is a superset of JavaScript that compiles to clean, readable JavaScript code.

What Is TypeScript?

TypeScript is an open-source programming language developed by Microsoft that builds on JavaScript by adding static type definitions. Types provide a way to describe the shape of objects, enabling better documentation and catching errors early in development.

Key Characteristics

  • Static Typing: Optional type annotations catch errors at compile time.
  • JavaScript Superset: All valid JavaScript is valid TypeScript.
  • Compiles to JavaScript: Transpiles to clean, readable JS for any browser/environment.
  • Enhanced IDE Support: Superior autocomplete, navigation, and refactoring.
  • Modern Features: Latest ECMAScript features plus additional TypeScript-specific features.
  • Gradual Adoption: Add types incrementally to existing JavaScript projects.
Note: TypeScript doesn't run directly in browsers ‚Äì it must be compiled to JavaScript first.

Why Use TypeScript?

TypeScript helps catch bugs before runtime, improves code maintainability, and enhances developer productivity with excellent tooling. It's become the standard for large-scale JavaScript applications.

Common Use Cases

Enterprise Apps Large Codebases React/Angular/Vue Node.js Backends API Development Library Development Team Projects Type-Safe Code Refactoring Documentation

Advantages

  • Early Error Detection: Catch bugs during development, not production.
  • Better IDE Experience: Intelligent autocomplete and inline documentation.
  • Refactoring Confidence: Safe, automated refactoring tools.
  • Self-Documenting Code: Types serve as inline documentation.
  • Team Collaboration: Clear contracts between code modules.
  • Industry Standard: Adopted by Microsoft, Google, Airbnb, Slack, and more.

Core Concepts

Basic Types

// Primitive types
let name: string = "DevSphere";
let age: number = 25;
let isActive: boolean = true;
let nothing: null = null;
let notDefined: undefined = undefined;

// Arrays
let numbers: number[] = [1, 2, 3];
let names: Array<string> = ["Alice", "Bob"];

// Tuples (fixed-length arrays with specific types)
let person: [string, number] = ["John", 30];

// Any (opt-out of type checking - use sparingly!)
let anything: any = "could be anything";

Interfaces & Types

// Interface
interface User {
    id: number;
    name: string;
    email: string;
    age?: number;           // Optional property
    readonly role: string;  // Read-only property
}

// Type alias
type ID = string | number;

// Using interfaces
const user: User = {
    id: 1,
    name: "John Doe",
    email: "john@example.com",
    role: "admin"
};

// Function with typed parameters and return
function greet(user: User): string {
    return `Hello, ${user.name}!`;
}

Functions

// Function with type annotations
function add(a: number, b: number): number {
    return a + b;
}

// Arrow function
const multiply = (a: number, b: number): number => a * b;

// Optional parameters
function greet(name: string, greeting?: string): string {
    return `${greeting || "Hello"}, ${name}!`;
}

// Default parameters
function createUser(name: string, role: string = "user") {
    return { name, role };
}

// Rest parameters
function sum(...numbers: number[]): number {
    return numbers.reduce((acc, n) => acc + n, 0);
}

Union & Intersection Types

// Union types (OR)
type Status = "pending" | "approved" | "rejected";
type ID = string | number;

function printId(id: ID) {
    console.log(`ID: ${id}`);
}

// Intersection types (AND)
interface Person {
    name: string;
}

interface Employee {
    employeeId: number;
}

type Staff = Person & Employee;

const employee: Staff = {
    name: "John",
    employeeId: 123
};

Generics

// Generic function
function identity<T>(value: T): T {
    return value;
}

const num = identity<number>(42);
const str = identity<string>("hello");

// Generic interface
interface ApiResponse<T> {
    data: T;
    status: number;
    message: string;
}

const userResponse: ApiResponse<User> = {
    data: { id: 1, name: "John", email: "john@example.com", role: "user" },
    status: 200,
    message: "Success"
};

// Generic class
class DataStore<T> {
    private data: T[] = [];
    
    add(item: T): void {
        this.data.push(item);
    }
    
    getAll(): T[] {
        return this.data;
    }
}

Classes

class Person {
    private name: string;
    public age: number;
    protected role: string;
    
    constructor(name: string, age: number, role: string) {
        this.name = name;
        this.age = age;
        this.role = role;
    }
    
    greet(): string {
        return `Hello, I'm ${this.name}`;
    }
    
    static species(): string {
        return "Homo sapiens";
    }
}

// Inheritance
class Developer extends Person {
    private language: string;
    
    constructor(name: string, age: number, language: string) {
        super(name, age, "developer");
        this.language = language;
    }
    
    code(): string {
        return `Coding in ${this.language}`;
    }
}

const dev = new Developer("Alice", 28, "TypeScript");
console.log(dev.greet());  // Hello, I'm Alice
console.log(dev.code());   // Coding in TypeScript

Type Guards & Narrowing

function printValue(value: string | number) {
    if (typeof value === "string") {
        // TypeScript knows value is string here
        console.log(value.toUpperCase());
    } else {
        // TypeScript knows value is number here
        console.log(value.toFixed(2));
    }
}

// Custom type guard
function isUser(obj: any): obj is User {
    return obj && typeof obj.name === "string";
}

if (isUser(someObject)) {
    // TypeScript knows someObject is User here
    console.log(someObject.name);
}

Ecosystem & Frameworks

Frontend Frameworks

  • Angular: Built with TypeScript from the ground up.
  • React: Excellent TypeScript support via Create React App.
  • Vue 3: Written in TypeScript, first-class support.
  • Next.js: React framework with built-in TS support.
  • Svelte: TypeScript support via preprocessors.

Backend Frameworks

  • NestJS: TypeScript-first Node.js framework.
  • Express + TypeScript: Type-safe Express apps.
  • Fastify: High-performance with TS support.
  • tRPC: End-to-end typesafe APIs.
  • Deno: TypeScript runtime (no compilation needed).

Development Tools

  • ts-node: Execute TypeScript directly.
  • tsx: Fast TypeScript execution.
  • ESLint + TypeScript: Linting for TS.
  • Prettier: Code formatting.

Type Definitions

  • DefinitelyTyped: Type definitions for JS libraries.
  • @types/*: npm packages with type definitions.
  • Built-in types: DOM, Node.js, etc.
