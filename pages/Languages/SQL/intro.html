What Is SQL?

SQL (Structured Query Language) is a standardized programming language specifically designed for managing and manipulating relational databases. Created in the 1970s, SQL has become the universal language for database interaction.

Key Characteristics

  • Declarative: You specify what you want, not how to get it.
  • Standardized: ANSI/ISO standard with wide adoption.
  • Set-Based: Operates on entire sets of data, not individual records.
  • Portable: Works across different database systems with minor variations.
  • Powerful: Handles simple queries to complex analytical operations.
  • English-Like: Readable syntax using keywords like SELECT, WHERE, JOIN.
Pronunciation: SQL can be pronounced as "S-Q-L" or "sequel" ‚Äî both are widely accepted.

Why Use SQL?

SQL is the backbone of data management in applications worldwide. From small websites to enterprise systems, SQL powers data storage, retrieval, and analysis.

Common Use Cases

Web Applications Data Analysis Business Intelligence E-commerce Banking Systems Healthcare Records Inventory Management Reporting Data Warehousing Analytics

Advantages

  • Universal: Works with MySQL, PostgreSQL, SQL Server, Oracle, and more.
  • Efficient: Optimized for working with large datasets.
  • Reliable: ACID compliance ensures data integrity.
  • Easy to Learn: Readable syntax similar to English.
  • In-Demand: Essential skill for many tech roles.
  • Mature Ecosystem: Decades of development and optimization.

Core Concepts

Tables (Relations)

Data is organized in tables with rows (records) and columns (fields):

-- Example: users table
+----+---------+-----+------------------+
| id | name    | age | email            |
+----+---------+-----+------------------+
|  1 | Alice   |  25 | alice@email.com  |
|  2 | Bob     |  30 | bob@email.com    |
|  3 | Charlie |  35 | charlie@email.com|
+----+---------+-----+------------------+

Basic SQL Commands

-- SELECT: Retrieve data
SELECT name, email FROM users;

-- WHERE: Filter results
SELECT * FROM users WHERE age > 25;

-- INSERT: Add new records
INSERT INTO users (name, age, email)
VALUES ('David', 28, 'david@email.com');

-- UPDATE: Modify existing records
UPDATE users SET age = 26 WHERE name = 'Alice';

-- DELETE: Remove records
DELETE FROM users WHERE id = 3;

SQL Statement Categories

DDL (Data Definition Language)
  • CREATE - Create tables/databases
  • ALTER - Modify structure
  • DROP - Delete tables/databases
DML (Data Manipulation Language)
  • SELECT - Query data
  • INSERT - Add data
  • UPDATE - Modify data
  • DELETE - Remove data

Practical Examples

Creating a Table

CREATE TABLE employees (
    id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    salary DECIMAL(10, 2),
    hire_date DATE,
    department_id INT
);

Inserting Data

-- Insert single record
INSERT INTO employees (first_name, last_name, email, salary, hire_date)
VALUES ('John', 'Doe', 'john.doe@company.com', 75000, '2024-01-15');

-- Insert multiple records
INSERT INTO employees (first_name, last_name, email, salary, hire_date)
VALUES 
    ('Jane', 'Smith', 'jane.smith@company.com', 80000, '2024-02-01'),
    ('Mike', 'Johnson', 'mike.j@company.com', 65000, '2024-03-10');

Querying with Conditions

-- Find high earners
SELECT first_name, last_name, salary
FROM employees
WHERE salary > 70000
ORDER BY salary DESC;

-- Filter with multiple conditions
SELECT *
FROM employees
WHERE salary BETWEEN 60000 AND 80000
  AND hire_date >= '2024-01-01';

-- Pattern matching
SELECT *
FROM employees
WHERE email LIKE '%@company.com';

Aggregation & Grouping

-- Count employees
SELECT COUNT(*) AS total_employees
FROM employees;

-- Average salary by department
SELECT department_id, 
       AVG(salary) AS avg_salary,
       COUNT(*) AS employee_count
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 70000;

Joining Tables

-- Inner Join: Get employee names with department names
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;

-- Left Join: Include employees without departments
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.id;

Popular Database Systems

MySQL

Most popular open-source database

  • Used by Facebook, Twitter, YouTube
  • Great for web applications
  • Fast read operations

PostgreSQL

Advanced open-source database

  • ACID compliant, highly reliable
  • Advanced features (JSON, arrays)
  • Excellent for complex queries

SQLite

Lightweight, file-based database

  • No server setup required
  • Perfect for mobile apps
  • Great for learning SQL

Microsoft SQL Server

Enterprise database system

  • Tight integration with .NET
  • Powerful analytics tools
  • Used in enterprise environments

SQL Dialects

While SQL is standardized, each database has slight variations:

  • MySQL: LIMIT, AUTO_INCREMENT
  • PostgreSQL: LIMIT, SERIAL, advanced data types
  • SQL Server: TOP, IDENTITY, T-SQL extensions
  • Oracle: ROWNUM, PL/SQL

Advanced SQL Concepts

Subqueries

-- Find employees earning above average
SELECT first_name, last_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

Indexes (Performance)

-- Create index for faster queries
CREATE INDEX idx_email ON employees(email);
CREATE INDEX idx_dept_salary ON employees(department_id, salary);

Transactions

-- Ensure data consistency
START TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;

COMMIT;  -- Or ROLLBACK to undo changes
